# 阶段③：指令系统与 Inline Keyboard（To C 友好）

## 1) Bot 指令清单（MVP + 可扩展）

### 普通成员
- `/start`：帮助与入群指引
- `/sign`：签到
- `/points`：查询积分

### 管理员（群内）
- `/admin`：打开管理面板（Inline Keyboard）
- `/ad`：发布一条广告（MVP：`/ad 标题|内容`）

> 可扩展：`/rules`（群规）、`/lottery`（抽奖）、`/warn`（警告）、`/mute`（禁言）、`/ad`（发布广告）等。

## 2) 指令权限说明

- `/start`：所有人
- `/sign`、`/points`：群内所有成员（各群独立计算）
- `/admin`：仅群管理员/群主

## 3) 核心菜单树结构（MVP）

> 说明：本文件为早期 MVP 草案，当前实际菜单与功能以 `docs/setup/06_feature_truth_table.md` 为准。

- 管理面板
  - 群设置（开关项）
    - 签到 开/关
    - 新人验证 开/关
    - 内容审核 开/关
    - 屏蔽链接 开/关
    - 广告 开/关
    - 商业化 开/关
  - 积分（说明页，后续可扩展发放/扣除）
  - 新人验证（说明页，后续可扩展题库/超时动作）
  - 内容审核（说明页，后续可扩展关键词、白名单、处罚）
  - 广告与订阅（说明页，后续可扩展投放/套餐）

## 4) 关键交互流程（MVP）

### A. 签到流程
1) 成员在群里发送 `/sign`
2) 机器人写入签到记录 + 积分流水 + 更新余额
3) 返回“签到成功 + 当前余额”

### B. 新人验证流程
1) 新成员入群触发事件
2) 机器人限制发言权限（需要 bot 有管理员权限）
3) 机器人发送带按钮消息“我不是机器人”
4) 新成员点击按钮 → 验证通过 → 解除限制

### C. 广告发布（骨架）
1) 管理员打开 `/admin` → “广告与订阅”
2) 未来扩展：创建活动、设置频次、预览并发布

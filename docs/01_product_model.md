# 阶段①：产品与功能模型（To C 群管理机器人）

## 1) 功能模块拆分（一级 / 二级）

### A. 群基础与配置
- 群绑定与多群隔离
- 群级配置（语言、开关、阈值、策略）
- 权限与角色（群主/管理员/普通成员）

### B. 新人入群与反垃圾
- 新人验证（按钮/算术题/关键词口令）
- 新人限权（限制发言/媒体/链接）
- 超时处理（继续禁言/踢出/提醒管理员）

### C. 内容审核与风控
- 链接/敏感词/刷屏检测
- 违规记录与黑名单
- 自动处置（删消息/警告/禁言/封禁）

### D. 积分体系与运营
- 签到（每日/冷却/连签可扩展）
- 积分账户与流水
- 积分兑换（抽奖/福利/权限）

### E. 抽奖与活动（可扩展）
- 抽奖活动（报名/开奖/风控）
- 活动模板（任务/排行榜）

### F. 广告与商业化（To C）
- 群内广告发布（管理员/广告主）
- 广告投放与频控（可扩展）
- 群级订阅与套餐（免费/付费）

### G. 多语言与可扩展
- i18n 字符串表
- 功能开关与灰度（群级/用户级）

## 2) 每个模块的使用场景

- **配置**：群主希望不用外部后台，在群内就能开关功能与策略调参
- **新人验证**：防止机器人/广告号入群刷屏，降低管理员负担
- **内容审核**：快速处理链接/关键词等常见违规，沉淀违规数据
- **积分运营**：通过签到/任务让群活跃，后续可用于抽奖、兑换、付费权益
- **广告商业化**：群主希望通过广告或订阅变现，机器人提供标准化能力

## 3) 管理员功能 vs 普通用户功能

- **管理员**
  - 群级配置：开关、阈值、策略选择
  - 新人验证策略选择与超时动作
  - 审核策略与处罚策略
  - 积分发放/扣除（可扩展）
  - 广告发布与投放（可扩展）
  - 订阅/套餐查看与续费（可扩展）

- **普通用户**
  - 签到、查积分
  - 参与抽奖/活动（可扩展）
  - 完成新人验证、查看规则

## 4) 必须支持的 MVP 功能列表

- 多群隔离 + 群级独立配置
- 群内 `/admin` 管理面板（Inline Keyboard）
- 签到积分（`/sign`）+ 查积分（`/points`）
- 新成员入群验证 + 限制发言 + 验证放行
- 基础内容审核（屏蔽链接）+ 违规记录
- 广告与订阅的数据模型与开关位（先骨架，便于迭代）

# 阶段⑤：商业化与扩展设计（To C 群级订阅）

## 1) 收费模型设计（群级订阅）

- **计费对象**：以“群”为单位订阅（同一用户可为多个群订阅）
- **结算周期**：月付 / 年付（年付折扣）
- **升级路径**：免费 → Pro → Business（或 Creator/Pro/Team）
- **支付渠道**：
  - Telegram Payments（支持多支付提供商）
  - 或外部支付（跳转链接/小程序/落地页），再回写订阅状态（需风控）

## 2) 免费版 / 付费版功能区分（建议）

### 免费版（Free）
- 基础签到积分、查积分
- 新人验证（基础按钮验证）
- 基础内容审核（屏蔽链接）
- 轻量配置（开关为主）

### Pro（付费）
- 关键词审核、白名单、正则规则
- 处罚策略可配置（删/禁言/封禁）
- 积分玩法增强（任务、排行榜、兑换）
- 广告系统（群内广告位、频控、投放统计）
- 数据报表（活跃、留存、违规）

### Business（更高档）
- 多管理员协作、审计日志
- API/插件化扩展
- 多群统一管理（平台化雏形）

## 3) 风控与滥用防护

- **反刷**：对签到/积分获取做频率限制；对同一用户多群薅羊毛做策略
- **反垃圾**：新人验证 + 链接/关键词拦截 + 违规累积处罚
- **防绕过**：管理员操作（删/禁言/封禁）必须校验 bot 是否具备权限并记录
- **支付风控**：支付回调签名校验；订阅状态变更留痕

## 4) 未来功能扩展路线图

- V1（MVP）：多群配置、签到积分、新人验证、基础审核、广告/订阅骨架
- V2：关键词审核、处罚策略、积分任务/排行榜、抽奖活动
- V3：广告投放系统（频控、统计、投放队列），订阅支付闭环
- V4：平台化（多群面板、插件系统、开放 API、数据分析）
